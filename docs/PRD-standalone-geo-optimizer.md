## 1. 背景与问题定义

### 1.1 现有工作流的局限

GEO Strategic Hub 的核心工作流分三步：**意图诊断 → 策略沙盘 → 内容生产**。这套流程适合从零开始构建 GEO 内容体系，但存在以下场景无法覆盖：

- 用户已有**现成内容**（官网文章、产品页、技术文档、数据表摘要），希望直接对其进行 GEO 强化
- 用户没有时间完成完整的三步诊断，只需快速优化一篇稿件
- 外部团队（代理商、合作方）需要使用工具，但无法接触诊断数据

### 1.2 目标

提供一个**独立的内容优化入口**，用户直接输入已有内容（粘贴文本 / URL / 上传文档），系统对其进行 GEO 优化并输出增强版本，全程**不依赖诊断流程**。

## 2. 用户画像与使用场景

### 2.1 主要用户

| 角色 | 场景 |
|------|------|
| 内容营销工程师 | 将产品页文案 GEO 化，提升 AI 检索引用率 |
| 技术写作者 | 强化已完成的技术白皮书或应用笔记 |
| 产品经理 | 快速优化竞品对比页，增加 AI 推荐权重 |
| 代理商 / 合作伙伴 | 无诊断权限但需使用 GEO 优化能力 |

### 2.2 典型使用场景

**场景 A — URL 优化**
> 用户粘贴公司官网产品页 URL → 系统抓取内容 → GEO 优化 → 输出改写版本 + 改善报告

**场景 B — 文档上传**
> 用户上传 PDF/TXT 规格书 → 系统解析 → 针对知乎长文格式优化 → 输出结构化文章

**场景 C — 粘贴文本**
> 用户将草稿直接粘贴 → 选择目标平台和 GEO 方法 → 一键强化 → 对比前后 GEO 信号改善

**场景 D — 多来源合并**
> 用户同时添加官网 URL + 数据表摘要 → 内容合并优化 → 输出综合强化版本

## 3. 功能需求

### 3.1 内容输入（核心需求）

| # | 功能 | 优先级 | 说明 |
|---|------|--------|------|
| F1 | 粘贴文本输入 | P0 | textarea 直接粘贴，点击"添加为来源"加入队列 |
| F2 | URL 抓取 | P0 | 输入 URL 自动抓取内容，显示来源名称和字数 |
| F3 | 文件上传 | P0 | 支持 `.txt`、`.md`、`.pdf` 文件，多文件多次添加 |
| F4 | 多来源管理 | P1 | 来源列表展示（名称、字数、类型），支持单独删除 |
| F5 | 多来源合并优化 | P1 | 多个来源内容合并后统一送入优化管道 |

### 3.2 优化配置

| # | 功能 | 优先级 | 说明 |
|---|------|--------|------|
| F6 | 优化方向说明 | P1 | 可选 textarea，用户描述目标受众、语气等额外要求 |
| F7 | 环境阵地选择 | P0 | 支持 10 个平台：知乎、微信、LinkedIn、Reddit 等 |
| F8 | 表达载体选择 | P0 | 支持 5 种格式：长文、短帖、对比矩阵、教程、Newsletter |
| F9 | GEO 方法选择 | P0 | 8 种方法，最多同时选 3 个，含推荐组合一键切换 |

### 3.3 输出功能（全部继承自 Step 3）

| # | 功能 | 优先级 | 说明 |
|---|------|--------|------|
| F10 | 流式内容输出 | P0 | 实时流式展示优化后内容，含 18,000 字符安全截断 |
| F11 | GEO 信号前后对比 | P0 | 量化数据、技术术语、可引用块、模糊语气词 before/after |
| F12 | GEO 优化分析 | P0 | 结构化说明具体改动和预期改善 |
| F13 | JSON-LD Schema 生成 | P1 | 自动生成 Schema.org 结构化数据，可复制 |
| F14 | 去 AI 味（Humanize） | P1 | 对输出内容二次处理，剔除 AI 痕迹 |
| F15 | 多语言翻译 | P1 | 支持翻译到中/英/日/韩 |
| F16 | 复制 / 导出 .md | P1 | 一键复制内容或导出 Markdown 文件 |
| F17 | Rate limit 倒计时 | P2 | API 429 自动重试，UI 展示倒计时 |

### 3.4 零幻觉约束（硬性需求）

- 输出内容中的每一个数据、产品名、规格参数**必须**来自用户提供的来源
- AI 不得凭空添加任何统计数字、价格或技术声明
- 如果 GEO 方法需要某类数据而来源中没有，**跳过该指令**而非捏造

### 3.5 模式切换

| # | 功能 | 优先级 |
|---|------|--------|
| F18 | Header 模式切换按钮 | P0 | 黄色高亮激活状态，切换隐藏/显示向导进度条 |
| F19 | 向导状态保持 | P0 | 切换到独立模式后再返回，诊断数据和策略选择完好 |
| F20 | 模式偏好持久化 | P2 | localStorage 保存上次的模式选择 |

## 4. 非功能需求

| 类别 | 要求 |
|------|------|
| 国际化 | 支持中文、英文、日语三种 UI 语言 |
| 性能 | 首字节流式输出 < 3s（受 Gemini API 影响） |
| 安全 | API Key 通过 Secret Manager 注入，不暴露在前端代码 |
| 兼容性 | 桌面浏览器（Chrome/Safari/Firefox），响应式适配 1280px+ |
| 内容限制 | 单次来源内容上限 8,000 字符送入 AI；输出流截断于 18,000 字符 |
| 错误处理 | API 失败显示错误信息；Rate limit 自动重试最多 3 次 |

## 6. 边界说明（Out of Scope）

- **不支持**：实时 URL 监控（定期重新抓取）
- **不支持**：批量文件队列处理（每次一个优化任务）
- **不支持**：用户账户 / 历史记录保存
- **不支持**：原文与优化版本的 diff 视图（当前只展示优化后版本）
- **不影响**：原有三步向导工作流（完全独立，切换后状态保持）

## 7. 成功指标

| 指标 | 目标 |
|------|------|
| GEO 信号改善率 | 量化数据指标平均提升 ≥ 3 个 |
| 用户完成率 | 从添加来源到获得输出的转化 ≥ 70% |
| 零幻觉合规 | 输出内容中未出现来源中不存在的数据 |
| 首次输出延迟 | ≤ 5s（含 Gemini API 冷启动） |
